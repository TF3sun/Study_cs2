# 🎮 C# .NET 기반 Up-Down 숫자 맞추기 게임

의존성 주입(DI) 컨테이너 구조와 xUnit 자동화 단위 테스트를 도입하여 결합도를 낮추고 유지보수성을 극대화한 현대적 C# 콘솔 Up-Down 게임 프로그램입니다.

---

## 🚀 주요 기능 및 규칙
- **Up-Down 게임 루프**: 프로그램이 생성한 난수(1~100)를 사용자가 맞추는 기본 게임 흐름을 제공합니다.
- **7회 시도 제한 (패배 조건)**: 사용자가 7회 초과로 정답을 맞히지 못하면 명확한 포맷의 실패 메시지를 출력하고 게임이 종료됩니다.
- **재시작 시스템**: 게임 승리 또는 패배 시 사용자의 선택(`y/n`)에 따라 안전하게 게임을 재시작하거나 종료합니다.
- **인코딩 버그 수정**: Windows 환경 콘솔에서 한국어 출력 시 문자열 앞에 물음표(`?`)가 붙어 나오는 인코딩(BOM) 깨짐 현상을 `Console.OutputEncoding` 설정을 통해 원천 차단했습니다.

---

## 🏗️ 아키텍처 및 설계 특징 (Architecture)

본 프로젝트는 SOLID 원칙 중 **의존성 역전 원칙(DIP)**을 준수하여 설계되었습니다.

### 1. 인터페이스 분리와 디커플링
- `GameManager` 클래스는 사용자 입출력을 담당하는 구체 클래스에 직접 의존하지 않고, `IUserController` 인터페이스에 의존합니다.
- UI 로직(콘솔)과 도메인 핵심 비즈니스 로직(게임 규칙)을 완전히 분리하여, 향후 GUI(WPF, WinForms)나 웹 환경으로 손쉽게 확장할 수 있습니다.

### 2. .NET Extensions DependencyInjection 활용
- `Microsoft.Extensions.DependencyInjection` 내장 DI 컨테이너를 사용하여 객체의 생명 주기를 자동으로 관리합니다.
- **수명 주기(Lifecycle) 디자인**:
  - `IUserController` (`AddSingleton`): 내부에 유동적인 상태(State)를 가지지 않으므로 싱글톤으로 등록하여 전역 자원 낭비를 방지합니다.
  - `GameManager` (`AddTransient`): 매 게임 실행 시마다 새로운 정답 난수와 시도 횟수 상태를 유동적으로 초기화하고 오염을 막기 위해 트랜지언트로 등록합니다.

## 🧪 자동화 단위 테스트 (Unit Test)

현업 표준에 맞춰 실제 실행 코드와 테스트 코드를 별도의 프로젝트(`Study_cs2.Tests`)로 분리하여 검증 환경을 구축했습니다.

- **테스트 프레임워크**: xUnit
- **모조 객체 라이브러리**: Moq

### 핵심 테스트 시나리오
1. **7회 초과 실패 및 포맷 검증**:
   - `Moq`를 이용해 사용자가 무조건 오답(`1`)만 연속으로 입력하는 가짜 유저 시나리오를 구성합니다.
   - 7회 시도 후 `"❌ 7회 동안 정답을 맞히지 못해 실패했습니다! 정답은 [...]였습니다."` 형태의 고정 포맷 메시지가 정확히 출력되는지 `It.Is<string>` 매칭 검사(StartsWith/EndsWith)를 통해 난수 값에 독립적으로 검증합니다.
2. **종료 수명 주기 검증**: 게임 패배 후 재시작 거부 시 `ShowExitMessage()`가 안전하게 호출되며 프로그램이 정상 종료되는지 추적합니다.
