Core API Reference

비즈니스 도메인 로직과 입출력 컨트롤러 환경이 독립적으로 격리된 Up-Down 게임 핵심 아키텍처 명세서입니다.

본 아키텍처는 Microsoft.Extensions.DependencyInjection 아키텍처 체계 내에서 설계 규격을 가집니다.

class GameManager Transient 수명 주기

Up-Down 게임의 핵심 비즈니스 로직, 난수 판정 제어, 승패 요건 및 전체적인 게임 루프 주기를 총괄하는 핵심 도메인 클래스입니다.

생성자 (Constructor)

public GameManager(IUserController userController)

의존성 주입(DI) 아키텍처 구조를 만족하기 위해 입출력을 추상화한 인터페이스를 주입받아 객체를 생성합니다.

파라미터명 타입 설명
userController IUserController UI 하위 제어를 위해 결합도를 차단한 상위 추상화 컨트롤러 객체

퍼블릭 메서드 (Public Methods)

void Run()

전체 게임 시퀀스 실행 주기를 구동합니다. 게임 시작 메시지 활성화, 1~100 난수 세팅, 시도 횟수 한계값(7회) 카운팅 루프 및 종료 처리를 통합 관리합니다.

interface IUserController Singleton 수명 주기

GameManager 내부 도메인 영역이 구체적인 물리 콘솔 장치나 UI 프레임워크에 종속되지 않도록 입출력 행위를 추상화 레이어로 격리하는 인터페이스입니다.

추상 메서드 명세 (Abstract Methods)

bool ShowStartMessage()

게임 진입 초기 인트로 안내 문구를 출력하고 게임 진행 의사 여부를 검증합니다.

Return: true 반환 시 게임 시퀀스 루프가 트리거되며, false 시 즉시 프로세스가 반환됩니다.

int RequestNumberInput()

사용자에게 숫자를 입력하도록 요구하고 정수 형태 데이터로 안전하게 변환하여 수집합니다.

Return: 사용자가 검증 필터를 통과하여 입력 완료한 1~100 사이의 유효 정수값.

void DisplayResult(string message, int count)

매 판정 루프 처리 결과를 정돈된 형태로 포맷팅하여 화면에 정형 출력합니다. (정답 성공 판정, 7회 초과 패배 확정 포맷 등 일괄 대응)

인자명 타입 설명
message string "업", "다운", "정답입니다!" 혹은 지정 실패 규격 포맷 데이터 문자열
count int 현재까지 누적 산정된 유효 시도 횟수 변수
bool AskForRestart()

게임 결과 시퀀스가 최종 종결(성공 정답 처리 혹은 7회 실패 확정)된 시점에서 재시작 의사(y/n)를 분기 처리합니다.

Return: true 반환 시 전체 자원이 재할당되어 새 게임이 셋업되며, false 시 종료 처리가 트리거됩니다.

void ShowExitMessage()

재시작 거부 혹은 강제 중단에 의해 프로세스가 완전 종료되기 직전, 정상 시스템 종료 안내 문구를 스크린에 마킹합니다.
